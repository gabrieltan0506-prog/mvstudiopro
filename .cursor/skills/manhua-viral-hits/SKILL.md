# 漫剧爆款情报 Skill

## 触发

用户给出：**漫剧/短剧链接**、平台榜单、或说「分析为什么火 / 爆款逻辑 / 对标某部漫剧」。

源笔记（名单与出处）：`docs/2026Jul17/manhua-viral-hits-2026h1.md`

## 爆款底层三逻辑（写作与选题时对照）

1. **产能与试错**：低成本高频出片，同一爽点模板可快速变体；工厂侧用编剧室 → 六段链路批量铺板，而不是单集精雕。
2. **IP / 类型势能**：成熟题材标签（边关罪妻、吞噬进化、游戏爽文）降低冷启动；选题优先「类型可识别 + 标题钩子一眼懂」。
3. **前 3 秒情绪**：开篇必须视觉钩子 + 冲突句；之后按网文节奏密铺打脸/逆袭/升级节点。

## 国内 vs 出海（选题时先定市场）

| 维度 | 国内 | 出海 |
|------|------|------|
| 格局 | 红海、分成、合规收紧 | 蓝海、高 ARPU 短剧 App |
| 题材 | 玄幻逆袭、种田穿越、古言边关、沙雕 | 狼人/吸血鬼、系统觉醒、本地化爽文 |
| 策略 | 多平台试数据；注意 AI 标识与低质限流 | 忌生硬搬国内 IP；按本地受众改设定 |

## 链接分析清单（按序写短结论）

对每条用户链接，输出：

1. **钩子**：标题/封面/前 3 秒在卖什么情绪  
2. **题材标签**：3–6 个中文关键词（可供工厂 topic）  
3. **爽点密度**：每 15–30s 是否有可视觉化的反转/升级  
4. **人设**：男女主欲望与压迫来源（一句话）  
5. **场景建议**：对齐 `recommendManhuaSceneFromTopic` 可命中的词（边塞/宗门/校园/末日等）  
6. **手法建议**：对齐 `recommendCraftShotFromTopic`（审讯/修仙/武侠/谍战等细项优先）  
7. **合规注意**：是否需 AI 标识、低质堆砌风险（国内）  
8. **可复制点 / 不可抄点**：只借结构与节奏，不成稿抄台词/画面/商标  

## 产出格式（贴回用户）

```text
【为何爆】一句话
【题材关键词】a / b / c
【建议场景】scene 倾向 + 关键词
【建议手法】craft 倾向 + 关键词
【工厂用法】把关键词填进 /canvas 工厂题材；铺板后可同步场景/手法
【风险】合规或同质化一句
```

## 映射到本仓库工厂（禁止另起模拟流水线）

| 外部「理想步骤」 | 本仓库实际 |
|------------------|------------|
| 剧本 + 分镜 | `/canvas` 编剧室 → story_brief / episode_beats；文本走 **Vertex** `geminiScript`（禁止改成 generativelanguage Key） |
| 静帧生图 | key_art / 导演中台静帧（既有链路） |
| 图生视频 | **Seedance Evolink**（`manhua:seedance-probe` 默认 SKIP；真打需显式 env；**不走 fal**） |
| 题材→场景/手法 | `shared/manhuaSceneAssetLibrary.ts`、`shared/craftShotBank.ts` |

## 硬规则

1. 前台成稿禁止写竞品名、供应商名、模型名、「仿写某某」。  
2. 情报数据以 docs 标注为准；未核实播放量不要当事实写进产品文案。  
3. 用户新链接的案例分析追加到 `docs/2026Jul17/manhua-viral-hits-2026h1.md`，必要时再开 PR 扩关键词库。  
4. 不要把外部模拟 TypeScript 流水线拷进 `scripts/` 当可执行代码。
