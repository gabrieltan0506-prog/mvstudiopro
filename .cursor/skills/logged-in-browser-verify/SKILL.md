# 登录态浏览器验收（人机接力）

用于 `mvstudiopro.com` 等**必须登录**才能看到的页面。Agent **不能**替你输密码；你登录，我抓取。

## 触发语（任一即执行本 Skill）

- 帮我登录后验 / 登录态回归 / 你登录我抓
- logged-in verify / 浏览器验收
- 验 #777 / 验 #778 / 验平台页线上（在未登录被挡时）

## 铁律

1. **禁止**让用户把密码/验证码发给聊天；只请用户在浏览器标签内完成登录。
2. **禁止**伪造「已登录」结论；未看到登录后 UI 不得报绿。
3. 前台回报遵守零技术泄漏：不写模型名/供应商（见 `frontend-no-tech-leak-always`）。
4. 抓取只读：默认不点付费按钮、不扣点；若验收必须点，先征得用户明文同意并说明会扣什么。

## 标准流程（必须按序）

### A. 开场（一次性说清）

用简体中文告诉用户：

1. 我将打开（或复用）Cursor 内置浏览器标签到目标 URL。
2. 请你在该标签完成登录（验证码/密码皆可）。
3. 登录成功后回复一句：**「已登录」**（或「登录好了」）。
4. 收到后我再锁标签、截图/快照、按清单核对并回报。

### B. 打开页面

1. `browser_tabs` list → 若已有 `mvstudiopro.com` 标签可复用，否则 `browser_navigate`。
2. 默认入口（按任务改）：
   - 平台 / 顾问 / 选题：`https://www.mvstudiopro.com/platform`
   - 画布 / 微调 / 工厂：`https://www.mvstudiopro.com/omni-canvas`（若 404 再试站点内实际路径）
3. `browser_snapshot`：若已在登录页 → 停，等用户登录；若已进工作台 → 跳到 D。

### C. 等待用户

- **停住工具调用**，等用户回复「已登录」。
- 超时或用户说登不上：记录 blockers，勿死循环刷页面。

### D. 登录后抓取

1. `browser_lock` → `browser_snapshot`（必要再 `browser_take_screenshot`）。
2. 确认已离开 `/login` 且可见产品主 UI；否则回报「仍像未登录」并请用户再试。
3. 按下方**任务清单**滚动/点开对应区块，只读采集文案与结构。
4. 结束后 `browser_lock` unlock。
5. 用简短中文回报：通过项 / 失败项 / 截图要点；并更新 `ACTIVE_TODO.md`（划掉完成、补新坑）。

## 常用验收清单（按需勾选）

### #778 平台顾问 + 选题初选

- [ ] 自定义创作区：选题初选**上方**有「人物背景与创作诉求」
- [ ] 背景为空时点「生成初选」→ 被拦截（toast/提示）
- [ ] 创作顾问：问虚拟资料类型/量大/利润/节点/定价 → **直接分块答**，不是「平台优先级/发帖排期」策略卡
- [ ] 回答中无模型名/供应商泄漏

### #777 画布微调

- [ ] 图片节点有「微调这张图」或等价入口
- [ ] 可见遮罩画笔 / 多图融合相关 UI（或「怎么用」提示）
- [ ] 失败提示不泄漏后端技术名

### #781 单一场景（合后）

- [ ] 编导区场景旁有「推荐单一」或推荐主场景文案
- [ ] 填仙侠类题材后出现推荐场景名

## 用户可复制的开场白（发给 Agent）

```text
登录态回归：我帮你登录。请打开平台页，我登完回「已登录」，你按 logged-in-browser-verify 抓 #778 清单。
```

## 不要做成什么

- 不要做成定时 Cursor Automation 指望云端带着你的 Cookie 登录（本流程依赖**本机 Cursor 浏览器会话**）。
- 不要在 Skill 里写死账号密码或验证码接口。
