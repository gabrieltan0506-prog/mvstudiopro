# MV Studio Pro — 全流程速度測算

## 一站式 AI 視頻生成全流程

### 流程步驟與預估時間

| 步驟 | 操作 | API 調用 | 預估時間 | 備註 |
|------|------|---------|---------|------|
| ① 生成虛擬偶像 | 選風格 + 性別 + 描述 → 生成圖片 | LLM generateImage (Forge AI) | **3-8 秒** | 免費引擎；付費 NBP 可能 5-15 秒 |
| ② 填入歌詞 | 用戶貼上歌詞文本 | 無 API 調用 | **5-10 秒**（用戶操作） | 純前端操作 |
| ③ 生成分鏡腳本 | LLM 分析歌詞 → 生成分鏡 + 預覽圖 | invokeLLM + generateImage × N | **30-90 秒** | 5 個分鏡並行生成預覽圖 |
| ④ 生成視頻 | 分鏡圖 → Kling 3.0 Omni I2V | Kling API createOmniVideoTask | **60-180 秒** | Std 5秒片段約 60s，Pro 10秒約 120-180s |
| ⑤ 拼接成品 | 多段視頻合併（如需要） | 後端 ffmpeg | **5-10 秒** | 服務端處理 |

### 最快全流程時間（理想情況）

| 場景 | 配置 | 總時間 |
|------|------|--------|
| **極速展示** | Forge AI + 3 分鏡 + Std 5s × 3 段 | **約 2-3 分鐘** |
| **標準品質** | Forge AI + 5 分鏡 + Std 5s × 5 段 | **約 4-6 分鐘** |
| **高品質成品** | NBP 4K + 5 分鏡 + Pro 10s × 5 段 | **約 10-15 分鐘** |
| **分鏡敘事模式** | Forge AI + 6 shot multi_prompt 15s | **約 2-3 分鐘**（單次 API 調用） |

### 關鍵瓶頸分析

1. **最大瓶頸：Kling 視頻生成**（60-180 秒/段）— 這是不可壓縮的，取決於 Kling API 的處理速度
2. **可優化環節：分鏡預覽圖生成** — 目前是並行生成，已經是最優方案
3. **最大加速點：3.0 Omni 分鏡敘事模式** — 一次 API 調用生成 6 個分鏡共 15 秒，比逐段生成快 5-6 倍

### 展示建議：2-3 分鐘速度演示

**最佳展示配置：**
- 虛擬偶像：Forge AI 免費引擎（3-5 秒）
- 歌詞：預填好一首短歌詞（0 秒）
- 分鏡：3 個分鏡（20-40 秒）
- 視頻：Kling 3.0 Omni 分鏡敘事模式，一次生成 15 秒（60-120 秒）
- **總計：約 90-165 秒 ≈ 1.5-3 分鐘**

**展示話術：**
> 「從零到一首完整 AI MV，只需要 2 分鐘。不需要攝影師、不需要剪輯師、不需要後期。一個人，一台手機，一首歌。」
