# AI 带货广告 Lite

这是一个只含 Markdown 文件的精简 Skill，用于上传到只允许 Markdown 文件的平台。它的目标不是替代完整版，而是让 Agent 先用最轻的说明启动工作。

## 优先路径

如果当前环境允许联网和执行命令，先安装完整版：

```bash
curl -fsSL https://raw.githubusercontent.com/huangbai-AI/ai-feed-ad-skill/main/install.sh | bash
```

安装后读取并使用：

```text
~/.codex/skills/ai-feed-ad-skill/SKILL.md
```

完整版包含脚本、模板、Loop、定时任务、结果报告和即梦 CLI 封装。能安装完整版时，不要把 Lite 当成最终实现。

## 无法安装时

如果不能联网、不能运行 shell、不能安装 Skill，使用 Markdown fallback：

- 读 [references/workflow.md](references/workflow.md) 执行主流程。
- 读 [references/templates.md](references/templates.md) 生成商品资料、脚本和视频提示词。
- 读 [references/quality-loop.md](references/quality-loop.md) 做重大错误检测、Loop 和结果报告。
- 读 [references/full-skill.md](references/full-skill.md) 给用户说明如何切回完整版。

## 基本规则

- 默认不抓取平台内容，不做高赞视频搜索。
- 只使用用户提供的商品图、商品信息、参考素材、本地素材、文档或明确给出的链接。
- 生产类型必须从有来源编号的小类中选；没有参考脚本依据的小类不要默认输出。
- 即梦 / Seedance 默认用 `seedance2.0fast`，商品图存在时优先多模态图生视频。
- 如果生成排队，只记录任务 ID 和状态，不重复提交。
- 质检只看重大错误，不因为钩子弱、CTA 弱、广告感不强而重试。
- Loop 只修失败点，保持原商品、人物、场景和脚本方向。

## 最小交付

每次至少给出：

1. 商品资料摘要。
2. 广告大类、小类和来源编号。
3. 10 秒或 15 秒脚本。
4. 可直接交给即梦 / Seedance 的视频提示词。
5. 重大错误检测表。
6. 结果报告，包含任务 ID、输出文件、质检结果、是否需要重试和下一步。


## 工作流摘要

# Markdown 主流程

## 1. 整理商品资料

从用户给的商品图、文字、视频或链接中提取：

- 商品名
- 品类
- 外观：颜色、形状、包装、材质、关键部件
- 目标人群
- 1 个主卖点
- 禁止编造的事实：价格、销量、认证、医疗功效

没有资料就标注“未知”，不要补编。

## 2. 选择广告类型

必须从以下有来源编号的小类中选：

- 剧情广告：情感反转剧情 `LD-01`
- 品牌大片：产品英雄广告 `LV-05`
- 品牌大片：电影感 TV 广告 `LV-06`
- 品牌大片：时尚生活方式跟拍 `LV-08`
- 达人带货：UGC 手持口播展示 `LV-01`
- 达人带货：开箱揭晓 `LV-02`
- 达人带货：使用场景证明 `LV-03`
- 达人带货：试穿前后变化 `LV-04`
- 达人带货：俯拍 ASMR 开箱 `LV-07`

飞书文档或素材里只出现名字、但没有完成脚本反推的小类，先作为待反推，不进入默认输出。

## 3. 写脚本

默认写 10 秒或 15 秒短视频。结构：

- `0-2 秒`：商品、结果、冲突或主视觉已经出现。
- `2-7 秒`：一个核心动作，证明主卖点。
- `7-10 秒`：商品定格或轻行动引导。

如果是 15 秒，把中段扩展为 2 个动作，但仍只讲 1 个主卖点。

## 4. 写即梦 / Seedance 提示词

提示词必须包含：

- 时长和比例：例如 `10 秒竖屏 9:16`
- 商品外观约束
- 人物、场景、动作
- 时间轴
- 字幕要求
- 负面要求

商品图存在时，建议 Agent 使用多模态生成；不能调用工具时，只输出提示词。

## 5. 生成和查询

如果能用完整版，交给完整版脚本处理。只用 Lite 时，给出建议命令，不要假装已经提交。

生成排队时：

- 记录任务 ID
- 记录模型
- 记录提示词文件或提示词正文
- 等待新结果
- 不重复提交同一条

## 6. 质检、Loop、报告

生成结果后读取 [quality-loop.md](quality-loop.md)。只检查重大错误。没有重大错误就进入交付或批量候选。

## 即梦提示词模板

```text
生成一条{时长}竖屏 9:16 带货广告。

商品是：{商品名}。
商品外观必须保持：{颜色、形状、包装、材质、关键部件}。
广告类型：{大类} / {小类}，来源编号：{来源编号}。
主卖点：{主卖点}。

时间轴：
0-2 秒：{开场商品、结果、冲突或主视觉}
2-7 秒：{核心动作和卖点证明}
7-10 秒：{商品定格或轻行动引导}

画面要求：
- 商品始终清楚可见。
- 人物、场景和商品前后一致。
- 字幕短、清楚、无乱码。

负面要求：
- 不改变商品颜色、形状、包装、材质和关键部件。
- 不要断手断脚、多手多指、脸崩、身体结构异常。
- 不要让商品消失、遮挡、变形。
- 不要编造价格、销量、认证、医疗功效。
```
