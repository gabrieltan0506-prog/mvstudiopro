# 钩子·半成品解法·咨询闭环 Skill

## 用途
所有平台选题文案、自定义选题、决策智库扩写默认启用。让用户停滑并想继续问你，而不是纯百科。

## 高度需求结构
1. **先内化**：从人设推断——客户是谁、核心痛点 1–2 个、半成品价值、观众会「解气/痛快」的**爽点**。
2. **标题**：好奇缺口 / 反常识 / 具体反差 / 时事或影视切口之一；忌正确但无聊；宜幽默，忌说教。
3. **钩子（黄金三秒）**：**痛点与爽点都要有一部分前置**——不是把完整答案甩完，而是抛出「刺痛 + 痛快苗头」，让人愿意看到最后才拿到完整收束。口语可出镜。禁止先背定义。
4. **正文**：
   - 开场：钩子（痛点火花 + 爽点预告）+ 点名「这类人」
   - 中段：痛点展开 + **半成品要点**（见下方「数目」）；爽点逐步兑现，仍留缺口
   - 结尾：把痛点/爽点**说完整**（收满那一口）+ **咨询/私信/预约/评论关键词** CTA
5. **分镜**：前 3 秒须同时出现痛点火花与爽点预告（同框或紧挨）；完整兑现留在末段；表情姿势可夸张（见 `cover-stop-scroll` / `vivid-anti-boring`）。

## 痛点 / 爽点：前置火花 → 最后收满（完播硬约束）
| 阶段 | 痛点 | 爽点 |
|------|------|------|
| **黄金三秒** | 点名窘境/误区一击（「就是我」） | 只给痛快**苗头**（反转一句、解气对比、数字钩子先抛一条） |
| **中段** | 展开代价与误区 | 半成品要点逐条兑现，仍故意留白 |
| **结尾** | 收束「原来卡在这里」 | **把爽点说完整**（最解气的一句/最后一招/全部清单齐了）+ CTA |

- **禁止**：痛点或爽点整段留到最后才第一次出现（前半像开会/背书，观众已划走）。
- **也禁止**：开头就把完整方案/全部爽点讲尽（没有「看到最后」的理由）。
- **合格手感**：前 3 秒「被戳到 + 想听完」；结尾「终于说满了 + 还想问你」。

## 数目很重要（完播结构）
- **宁少勿滥**：优先 **2–3 个**半成品要点；需要清单时最多 **5 个**，禁止一上来「十条妙招」堆墙。
- **数字钩子拉完播**：例如「五个妙招——先说第三个」：开头抛出最反常识/最想听的那一条（爽点前置），其余顺序展开，**最后才把五条说齐**。
- 列表标题要具体（「高压局里养精力的 3 个动作」），不要空喊「干货满满」。

## 桥段与反差（可与 `cultural-diversity` 叠用）
- **影视取材**：健康/精力/压力相关剧情可改写。例：《藏海传》藏海如何在权谋高压中维持精力旺盛——开场先抛最反常识的 1 点，**3 个要点说到齐**再 CTA。
- **古今人物**：任何时代均可（含宋朝范仲淹、文天祥、王安石等）；典籍与平民贵族皆可。
- **新创反差**：鼓励模型创造桥段选题，如「博士生去应徵服务员」式身份错位标题；不限时代。
- **禁止**：把完整可替代咨询的方案写尽；纯剧透墙无半成品。

## 强烈建议避免
- 纯百科堆砌、无受众、无 CTA
- 说教训话、「今天我们来聊聊」
- **痛点/爽点整段留到最后才出现**；或开头就说满、中段注水
- 要点数目失控（七八条以上说明书）
- 空壳「创作者/博主」自称（除非用户勾选放开）

## 强监管赛道（健康/医学/法律/金融等）
- 学者向表达；少病名/疗效承诺/临床强锚点
- 变现用「深度交流/私人顾问/体系课」，避免「线上问诊开药」
- 器官/慢病观察可走 `crossover-popsci`，仍须审核友好
