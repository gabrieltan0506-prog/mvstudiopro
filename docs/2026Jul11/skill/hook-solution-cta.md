# 钩子·半成品解法·咨询闭环 Skill

## 用途
所有平台选题文案、自定义选题、决策智库扩写默认启用。让用户停滑并想继续问你，而不是纯百科。

## 高度需求结构
1. **先内化**：从人设推断——客户是谁、核心痛点 1–2 个、你能给的半成品价值。
2. **标题**：好奇缺口 / 反常识 / 具体反差 / 时事切口之一；忌正确但无聊。
3. **钩子**：第一屏停滑句，口语可出镜。
4. **正文**：
   - 开场：钩子 + 点名「这类人」
   - 中段：痛点 + **2–3 个半成品要点**（给方向、故意留白完整方案）
   - 结尾：**咨询/私信/预约/评论关键词** CTA
5. **分镜**：至少一格痛点共鸣、一格半成品解法、末段引导交流；写清运镜/灯光/情绪。

## 强烈建议避免
- 纯百科堆砌、无受众、无 CTA
- 一次把完整可替代咨询的方案写尽
- 把用户写成空壳「创作者/博主」
- **正确但无聊**：论文摘要腔标题、无物件无动作的「××观/××管理」、六条同一模板换词（详见 `vivid-anti-boring`）

## 强监管赛道（健康/医学/法律/金融等）
- 学者向表达；少病名/疗效承诺/听诊器 CT 等临床强锚点
- 变现用「深度交流/私人顾问/体系课」，避免「线上问诊开药」
- 器官/慢病观察可走跨界拟人叙事（见 `crossover-popsci`），仍须审核友好
